# Documentation Review Checklist

## Code Documentation

- [ ] Public APIs have doc comments
- [ ] Complex algorithms explained
- [ ] Non-obvious "why" commented (not just "what")
- [ ] TODO/FIXME includes ticket reference
- [ ] Component usage and props documented
- [ ] Configuration options documented

## Change Documentation

- [ ] CHANGELOG entry for notable changes
- [ ] Breaking changes clearly documented
- [ ] Migration guide for breaking changes
- [ ] README updated if needed
- [ ] Infrastructure changes documented

## API & Interface Documentation

- [ ] Endpoint descriptions provided
- [ ] Request/response schemas documented
- [ ] Error responses documented
- [ ] Examples provided
- [ ] Input/output variables described (IaC)
- [ ] Required permissions/roles documented
